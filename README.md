# CARE 助理 — 关心你的每一个想法

> 基于 [lark-cli](https://github.com/larksuite/cli) 的个人效率助理，参加 [飞书 CLI 创作者大赛](https://waytoagi.feishu.cn/wiki/R4S3w8wTTie04nkYiL6c8rxon4d)。

**说一句话就记录，AI 自动分类，定时整理，待办变飞书任务。**

```
CARE = Capture 随手记 + Auto-sort 自动分 + Review 定期回顾 + Execute 照着做
```

## 这是什么

你是不是也经常这样——走在路上突然想到一个好点子，或者在群里看到一篇好文章，想着"回头整理一下"，然后就忘了。

CARE 助理解决这个问题。它是一个 Claude Code Skill，你只要开口说一句话，它就会自动帮你：

1. **记录** — 把你的话写入飞书多维表格
2. **分类** — AI 自动判断是待办、灵感还是文章，打上标签
3. **整理** — 定时批量处理未整理的记录
4. **执行** — 待办自动创建飞书任务，带截止日期和提醒

## 快速开始

### 前置条件

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code) 已安装
- [lark-cli](https://github.com/larksuite/cli) 已安装并配置好飞书应用
- 飞书应用需要开通：多维表格、云文档、任务 相关权限

### 安装

```bash
# 1. 克隆仓库到你的 skills 目录
git clone https://github.com/asiyoua/care-assistant.git
cp -r care-assistant ~/.claude/skills/care-assistant

# 2. 创建配置目录
mkdir -p ~/.care-assistant

# 3. 首次使用时，在 Claude Code 中说任何话即可自动触发 Setup
#    例如："帮我记一下明天开会"
```

首次使用时，CARE 助理会自动：
- 创建飞书多维表格（含记录表 + 文章收藏表）
- 创建所有必要的字段和视图
- 保存配置到 `~/.care-assistant/config.json`

## 使用方式

### 随手记 (Capture)

```
你：明天上午十点有个会
AI：已记录 [待办]：明天上午十点开会（已创建飞书任务）

你：灵感 做一个自动整理录音的工具
AI：已记录 [灵感]：做一个自动整理录音的工具

你：这篇不错 https://example.com/article
AI：已收藏：example.com/article
```

说任何包含"时间 + 动作"的句子都会自动触发记录。也可以用触发词：
`记录`、`待办`、`灵感`、`记一下`、`帮我记`、`随手记`

### 定期回顾 (Review)

```
你：回顾一下
AI：（生成飞书回顾文档，含概览、待办跟进、灵感回顾、方向观察、下期建议）
    回顾文档已生成：https://...
    - 本周 15 条记录，3 条待办已完成，5 条灵感待行动
    - 过期待办 2 条，建议关注
```

可以配合 Claude Code 的 Cron 定时任务，实现每周自动回顾。

### 录音转写 (Recording)

```
你：处理录音 /path/to/transcript.txt
AI：录音处理完成：
    - 表格记录 10 条（待办 3 / 灵感 4 / 其他 3）
    - 飞书文档已创建：https://...
```

录音转写会生成三层输出：
1. **多维表格记录** — 提取的待办、灵感、关键信息
2. **飞书文档** — 精炼提炼（金句 callout + 写作话题 + 行动要点表格）+ 原文存档
3. **飞书任务** — 待办自动创建

### 查看记录

```
你：查看记录 / 有多少待办
AI：（从多维表格读取并汇总展示）
```

## 架构

```
用户 ──说话──▶ Claude Code (Skill)
                    │
         ┌──────────┼──────────┐
         ▼          ▼          ▼
    飞书多维表格   飞书文档    飞书任务
    （数据存储）  （录音提炼）  （待办执行）
```

### 数据结构

**记录表** — 日常记录、待办、灵感

| 字段 | 类型 | 说明 |
|------|------|------|
| 记录标题 | 文本 | 内容摘要 |
| 来源 | 单选 | 终端 / 飞书对话 / 录音 |
| 标签 | 多选 | 待办 / 灵感 / 其他 / AI / ... |
| 详细内容 | 文本 | 补充说明 |
| 创建日期 | 日期 | 记录创建日期 |
| 截止日期 | 日期 | 仅待办类使用 |
| 完成状态 | 单选 | 未完成 / 已完成 |
| 关联文档 | 链接 | 飞书文档 URL / 本地路径 / 文件名 |

**文章收藏表** — 链接收藏（与记录表分开，不混在一起）

| 字段 | 类型 | 说明 |
|------|------|------|
| 标题 | 文本 | 文章标题或链接描述 |
| 链接 | URL | 文章链接 |
| 备注 | 文本 | 补充说明 |
| 标签 | 多选 | AI / 效率 / 产品 / 技术 / 商业 / 其他 |
| 来源 | 单选 | 终端 / 飞书对话 / 浏览器 |
| 收藏时间 | 日期 | 收藏时间 |

## 设计亮点

### 1. 零摩擦记录

不需要打开任何 App，在终端里说一句话就行。AI 自动判断内容类型、提取时间、打标签。

### 2. 文章与记录分离

文章链接写入独立的"文章收藏"表，不跟待办和灵感混在一起。各自有合适的字段和视图。

### 3. 智能去重

写入前自动读取已有记录，语义去重。不会因为重复表述产生冗余记录。

### 4. 录音三层输出

录音转写不只是简单存文本，而是产出：
- 结构化记录（可筛选、可追踪）
- 精炼文档（金句 + 写作话题 + 行动要点）
- 原文存档（可折叠，备查）

### 5. 标签系统

使用多选标签代替单选分类。一条记录可以同时是"待办"和"AI"，灵活组合。你可以自定义标签选项来适配自己的场景。

## 自定义

### 标签

修改 SKILL.md 中的标签选项和 `references/setup.md` 中建表时的 options。当前默认标签：

- 类型标签：待办、灵感、其他
- 主题标签：AI、社群、内容创作、线下

### 代理

如果 lark-cli 连接飞书 API 需要代理，在 SKILL.md 的命令前加上：

```bash
https_proxy=http://127.0.0.1:7897 http_proxy=http://127.0.0.1:7897
```

### 定时整理

配合 Claude Code 的 Cron 功能：

```bash
# 每周日晚 9:00 自动回顾
cron: 每周日 21:00 执行 "回顾一下"
```
