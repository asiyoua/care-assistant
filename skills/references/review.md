# Review — 回顾整理

## 触发

用户说"回顾"、"整理"、"周回顾"、"看看这周做了什么"。

## 流程

### 1. 读取记录

```bash
lark-cli base +record-list \
  --base-token <base_token> \
  --table-id <table_id> \
  --limit 200
```

同时读文章表（`article_table_id`）。按用户指定时间范围筛选，默认最近 7 天。

### 2. 分析记录

对记录进行分析：
- **待办**：检查完成状态，标注已过期/进行中/已完成
- **灵感**：按主题分组，点评进展和下一步
- **其他**：归纳有价值信息
- **标签分布**：发现用户关注的方向

### 3. 生成飞书回顾文档

```bash
lark-cli docs +create \
  --title "CARE 周回顾：MM.DD - MM.DD" \
  --folder-token <folder_token> \
  --markdown "<markdown content>"
```

文档结构：

```
## 概览
新增记录 N 条（待办 X / 灵感 Y / 其他 Z）、新增文章 P 篇、已完成待办 M 条

## 待办跟进
表格：待办 | 截止日期 | 状态 | 关联

## 灵感回顾
按主题分组，每条附点评

## 方向观察
标签分布分析

## 下期建议
2-3 条可执行建议
```

### 4. 汇报

```
回顾文档已生成：{doc_url}
- 本期 N 条记录，X 条待办已完成，Y 条灵感待行动
- 过期待办 Z 条，建议关注
```

## 整理模式

用户说"整理一下"时，除了生成回顾文档，还要：
1. 检查标签是否准确，不准确则更新（`+record-upsert` 只改标签字段）
2. 待办且未设截止日期的，根据内容推断或标记提醒
3. 待办且有过期截止日期的，更新完成状态为"进行中"
