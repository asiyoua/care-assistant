# Recording — 录音转写

## 触发

用户说"处理录音"、给了转写文档、或"分析转写"。

## 文件管理

- 存放目录：`~/.care-assistant/recordings/`
- 命名：`YYYY-MM-DD_主题描述.docx`
- 不在该目录的文件先复制过去并重命名

## 三层输出

### 第一层：多维表格记录

1. 读取转写内容
2. 读取已有记录（`+record-list --limit 200`）用于去重
3. 提取可记录项：
   - 待办事项（有行动要求）
   - 灵感/想法（点子、计划、构想）
   - 值得记录的信息（知识、经验、决策）
4. 提取后去重：新记录间互比合并，再与已有记录对比
5. 逐条写入记录表，`来源` = `录音`，`关联文档` 先留空
6. 待办类创建飞书任务

写入格式同 capture.md 的记录流程，但：
- `来源` 固定为 `录音`
- `关联文档` 先留空，第二层完成后回填

### 第二层：飞书文档

```bash
lark-cli docs +create \
  --title "录音：<主题描述>" \
  --folder-token <folder_token> \
  --markdown "<content>"
```

文档结构：

```
## 精炼提炼

### 亮点摘录
金句用 callout 高亮块

### 写作话题
每个话题含：切入点、参考段落、适合平台

### 行动要点
表格：行动项 | 优先级

---

## 原文转写
（完整内容，保持原样）
```

创建后回填：更新所有录音记录的 `关联文档` 字段为该文档 URL。

### 第三层：汇报

```
录音处理完成：
- 表格记录 N 条（待办 X / 灵感 Y / 其他 Z）
- 飞书文档已创建：{doc_url}
- 录音文件已归档：~/.care-assistant/recordings/YYYY-MM-DD_主题.docx
```

## 提取原则

- 口语化 → 简洁标题
- 同话题多句 → 合并为一条
- 明确行动 → 待办，模糊想法 → 灵感
- 金句/写作话题 → 放飞书文档不放表格
- 闲聊/重复/语气词 → 不记录
- 宁可标"其他"也不遗漏有价值信息
